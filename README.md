# 机器学习驱动的多因子选股系统

## 项目简介

本系统基于机器学习方法（XGBoost）构建多因子选股模型，以沪深300指数成分股为投资标的，融合技术因子与估值因子，通过滚动窗口训练与月度调仓实现量化选股策略。系统提供完整的数据获取、因子计算、模型训练、回测评估全流程自动化功能。

## 系统架构说明

系统采用模块化分层架构，自底向上分为四个层次：

### 数据层

- **数据源接口**：通过量化数据服务商API获取沪深300成分股列表、日线行情、换手率、估值指标
- **本地缓存模块**：将首次下载的数据序列化保存为本地文件，后续运行直接读取缓存
- **数据校验模块**：检查字段完整性、交易日连续性，处理缺失值与异常值

### 计算层

- **技术因子计算模块**：滚动计算不同周期的收益率、波动率、均线比值、成交量变化率、换手率波动、振幅
- **估值因子整合模块**：将PE、PB、PS、总市值等估值指标合并到日线数据
- **月频聚合模块**：取每月最后一个交易日数据作为当月特征，计算下月收益率并转换为二分类标签

### 模型层

- **滚动训练模块**：按月迭代，使用过去6个月数据训练，当前月数据测试
- **分类器模块**：XGBoost二分类模型，预测股票下月上涨概率
- **特征重要性评估模块**：输出各因子重要性排行

### 策略层

- **选股模块**：每月选取预测概率最高的20只股票
- **组合构建模块**：等权重配置，模拟月初买入、月末卖出
- **绩效评估模块**：计算累计收益、年化收益、最大回撤、夏普比率等

## 接口文档

系统设计为可直接运行的脚本程序，各模块间的数据传递约定如下：

| 功能模块 | 输入 | 输出 | 格式 |
|---------|------|------|------|
| 股票池获取 | 日期 | 沪深300成分股代码列表 | 字符串列表 |
| 日线行情 | 股票代码、起止日期 | 开高低收、成交量等 | DataFrame |
| 换手率与估值 | 交易日期 | 换手率、PE、PB、PS、总市值 | DataFrame |
| 因子计算 | 日线数据 | 增加15个因子列 | DataFrame |
| 月频聚合 | 日线因子数据 | 月末快照+下月标签 | DataFrame |
| 模型训练 | 训练集 | XGBoost模型对象 | 内存对象 |
| 预测输出 | 测试集 | 上涨概率列表 | 浮点数列表 |
| 选股与收益 | 概率、实际收益 | 当月组合收益率 | 浮点数 |
| 绩效汇总 | 月收益序列 | 各项绩效指标 | 数值+图表 |

### 关键配置参数

| 参数 | 含义 | 默认值 | 建议范围 |
|------|------|--------|----------|
| 训练窗口长度 | 历史月份数 | 6个月 | 3-12 |
| 选股数量 | 每月买入股票数 | 20只 | 5-50 |
| n_estimators | 决策树数量 | 500 | 100-1000 |
| max_depth | 最大树深 | 10 | 3-15 |
| learning_rate | 学习率 | 0.1 | 0.01-0.3 |
| subsample | 样本采样比例 | 0.8 | 0.5-1.0 |
| colsample_bytree | 特征采样比例 | 0.7 | 0.5-1.0 |

## 部署指南

### 环境要求

- 操作系统：Windows 10/11、macOS 11+、Linux
- Python：3.8及以上
- 内存：至少8GB，建议16GB
- 硬盘：至少5GB可用空间
- 网络：首次运行需联网下载数据

### 安装步骤

1. **安装Python环境**（推荐使用Anaconda或Miniconda）

2. **安装依赖库**（命令行执行）：
pip install pandas numpy scikit-learn xgboost tushare matplotlib


3. **配置数据接口令牌**：在代码中填入有效的Tushare token

4. **设置工作目录**：将脚本保存到本地文件夹，确保有读写权限

### 首次运行说明

- 首次运行自动下载数据并缓存，耗时约1-2小时
- 后续运行直接从缓存加载，耗时5-10分钟
- 如需更换回测时间段，修改起止日期后删除缓存文件重新运行

### 常见问题处理

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| API返回空数据 | token无效或接口变更 | 检查token，确认服务可用 |
| 缺失值过多报错 | 股票长期停牌 | 放宽缺失容忍度或剔除该股票 |
| 内存不足 | 数据量过大 | 减少回测区间或升级硬件 |
| 结果与报告不符 | 参数被修改 | 恢复默认参数，删除缓存重跑 |

## 使用手册

### 基本使用流程

1. 在脚本顶部配置区域填入有效token
2. 根据需要修改回测参数（可保持默认）
3. 执行主程序文件
4. 查看控制台输出的绩效指标和因子重要性图表

### 输出结果解读

- **月度收益明细表**：每月的日期、选股数量、组合平均收益率
- **核心绩效指标**：累计收益率、年化收益率、最大回撤、夏普比率、月度胜率
- **因子重要性排序图**：各因子在模型中的相对重要性

### 参数调整建议

| 目标 | 调整方式 |
|------|----------|
| 提高收益率 | 增加选股数量或缩短训练窗口 |
| 降低波动和回撤 | 增大训练窗口或选股数量 |
| 提升预测准确度 | 增加决策树数量或降低学习率 |
| 加快训练速度 | 减少决策树数量或剔除低重要性因子 |

### 注意事项

- 频繁修改参数后重跑不会重复下载数据，但模型训练仍需时间
- 缓存数据不会自动更新，使用新数据需删除缓存文件
- 回测结果仅供参考，不代表未来收益

### 扩展使用

- **更换股票池**：修改代码中的指数代码（如中证500）
- **增加新因子**：在因子计算函数中添加新列，并加入特征列表
- **更换模型**：替换XGBoost为其他分类器（如RandomForest、SVM）
